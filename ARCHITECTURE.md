# Architecture Navigation Map

## The Setup
This application is strictly focused on executing the **Blueprint Configurator**, a high-value utility that captures client diagnostic data to output a formal project brief. Previous marketing UI surfaces have been eliminated to rigidly enforce this objective.

## 1. Data Layer (`/src/data`)
Pure constants, static records, and schema definitions live here. Do not muddy React components with hardcoded data. 
- `configurator-content.ts`: Heavy strings, text copies around the blueprint logic
- `blueprint-options.ts`: The definitive record of select array options, archetype objects and personality tokens used across the Configurator steps.

## 2. Style Layer (`/src/styles`)
The styling architecture has intentionally been split out of a monolithic `index.css` file into purposeful partials.
- `index.css`: The barrel file. It solely imports other CSS partials in sequence. Do NOT place custom CSS declarations here.
- `base.css`: Root variables (Tailwind's `theme()` definitions, hex colours) and global `body` resets.
- `typography.css`: Typeface definitions (`@font-face` blocks for Raela/Nohemi).
- `configurator.css`: The core of the utility UI. Contains the `cf-surface` token logic defining states (base, elevated, subtle, selected), form control overrides, and glass card treatments.
- `animations.css`: Keyframe definitions and motion utilities.

## 3. UI Primitives (`/src/components/ui` & `/src/components/configurator/ui`)
Generic, dumb UI components. These should never fetch their own context or hold complex business state. 

**Global (`/src/components/ui/`):**
- System-wide atomic elements (Buttons, Inputs, Dialogs, Selects). These are generally generated by `shadcn/ui`.

**Domain Specific (`/src/components/configurator/ui/`):**
- Dumb components tailored specifically for the Configurator’s visual aesthetic. 
- Examples: `ColorWheelDiagram.tsx`, `InteractiveColorWheel.tsx`, `ReferenceGridItem.tsx`, `UserDetailsForm.tsx`.

## 4. Feature Flow (`/src/components/configurator/steps`)
The orchestrators. These components act as pages or "Steps" inside the Blueprint wizard.
- They maintain complex local state (drag loops, validation logic).
- They assemble the UI Primitives to create the layout.
- They dispatch the final data updates back to the master store or root parent.
- **Rule of Thumb:** If a Step component exceeds ~300 lines due to inline JSX elements, those elements must be abstracted into Domain Specific UI Primitives.

## 5. Animation Context (`framer-motion`)
We use `framer-motion` extensively for cinematic, scroll-linked interactions. 
- **Sticky Cards**: Use `position: sticky` over JS-based clamping. Wait to apply `will-change: transform` unless actively encountering scroll stutter, as indiscriminate will-change hints break intersection observers and `z-index` stacking contexts.
- **Performance**: High-intensity visual effects (like glass blur boundaries) use gradient masks (`src/components/cinematic/GlassEdges.tsx`) instead of `backdrop-blur()` filters to save GPU render budgeting.

## 6. The Run Engine (`vite`)
Always interact with the application through `npm run dev`. Avoid hot-reloading crutches; if styling begins to misbehave and Tailwind classes refuse to compile, restart the `vite` server before assuming code syntax failure.
